Open Design Questions

Decisions needed before Frontend Developer implements

Q1 — Animal Image Treatment
The card image areas currently show a placeholder letter. Final version: (A) illustrated artwork per species, (B) photo from camera capture, or (C) stylised silhouette + colour gradient? This affects AnimonCard layout significantly.
Q2 — Tab Icons
Currently using Unicode symbols (⊞◈◉◐). Should these be replaced with custom SVG icons or a proper icon library? The amber LED glow effect from the design spec requires specific icon shapes.
Q3 — Camera Screen Purpose
The prototype shows the camera as a scanning/detection tool. Confirm: (A) user points camera at real animal → AI identifies it, or (B) user manually enters details after photo, or (C) both? This determines the camera UI flow.
Q4 — Microinteractions Priority
4 microinteractions are defined in the spec but not yet implemented: Capture Stamp (new log animation), Scanner Lock-On (camera reticle pulse), Tab LED Activation, Shimmer Reveal (on card load). Which should be implemented first?
Q5 — Discover vs. Anílog Screen Distinction
Currently Discover = global/nearby animons feed, Anílog = personal collection. Should Discover show a map view, a community feed, or just nearby detections? This changes the Discover screen layout substantially.
