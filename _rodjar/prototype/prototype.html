Design Directions — Answered

Owner's input · March 5, 2026 · Reflected in this prototype

✅
CORE DIRECTION — Visual Reset
Full pivot from naturalist skeuomorphic to clean modern digital. White/neutral base, cool grey background (#F1F4F9), clean white cards. ALL colour from type chips + rarity badges + card image gradients. New font: Plus Jakarta Sans (modern, friendly) + Space Mono (data). The dark card footer strip is preserved — evolved as the Anílog signature, now clean digital instead of parchment.
✅
Q1 — Animal Image Treatment
Two modes: (A) cropped face photo from user's camera capture, (B) AI-generated Pokémon-style illustration per species. Card prototype uses type-derived gradient as the placeholder — this also serves as the fallback when neither is available.
✅
Q2 — Tab Icons + Camera Entry
Unicode icons kept for now. Camera FAB (floating action button, indigo gradient) added in the tab bar centre — elevated above the bar, clearly the primary action. The camera was previously invisible — this solves it.
✅
Q3 — Camera + Catching Mechanic
AR scanning: user points camera at real animal → AI identifies species, type, breed, rarity. Detection overlay shows name, species label, type chips, rarity, confidence %. Catching mechanic: swipe-up gesture (shown in prototype as animated arrow + label). Glossy detection shown visually on the overlay. Full catching animation (throw mechanic) is a V2 feature.
✅
Q4 — Stamps & Logbook Screen
New Logbook tab replaces Milestones in the main nav. 3-column grid, filtered by animal family (Canine, Reptile, Avian, Insect, etc.). Captured = full colour image + stamp count badge. Not captured = greyscale + "???". Stamp number = total catches of that species. Completion progress bar at top. Milestones moved to Profile → Milestones row item.
✅
Q5 — Social Feed + Trading
Discover = social feed (posts with optional embedded cards) + nearby sightings as horizontal scroll. Trading visible on: post cards ("Offer Trade"), Animon Detail page ("Offer for Trade" CTA), Profile ("Trade History" row). Full trade flow TBD — see Q8 below.
✅
Q6 — Brand Accent Colour
Grey. UI chrome (active tabs, FABs, stamps, progress, trade CTAs) uses slate-500 (#64748B) and slate-700 (#334155). This means the UI completely steps back and everything that pops is game content — type chips, rarity badges, card image gradients. No colour in the chrome means no favouritism toward any animal type.
✅
Q7 — Trade Flow Model
V1 Trade: request directly from a post on the Discover feed ("Offer Trade" button). The receiving user gets a notification and can accept/decline. Full async offer board is a V2 feature. Also confirmed: "Bestiary" renamed to "Logbook" throughout — and the ??? hidden names are intentionally retained as a discovery mechanic.
✅
Q8 — AI Image Flow
Option C confirmed. Flow: (1) user takes photo, (2) capture animation plays — stamped into the Logbook, (3) AI illustration is generated and shown as the primary card art. On Animon Detail, a toggle pill lets the user switch between "✦ AI Art" and "📷 My Photo". Cards consistently display AI art in the collection, preserving a unified aesthetic. Photo remains in the log entry as field data.
